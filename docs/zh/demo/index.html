        这套 Demo 重点想说明什么？

        • 不是付款能力：重点不在“支付成功”，而在“这笔钱为什么值得花”。
        • 不是自由消费：所有请求都要经过预算、权限和风险检查。
        • 不是黑箱决策：系统要明确说明“谁在做什么检查、为什么通过或拒绝”。
        • 不是只讲概念：每次评估都会生成一个可追溯的账本条目。
        说明：这里的阈值、白名单和风控规则都是示例策略，用于展示结构。 真实部署时，应根据任务风险、预算规模与合规要求进行配置，并引入人工审批/多签等机制。
